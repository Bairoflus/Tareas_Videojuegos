Git y GitHub

Git es una herramienta que utilizo para controlar versiones de los proyectos en los que trabajo mientras que GitHub me permite compartir esos proyectos y colaborar con otras personas en línea y de forma remota.

Usos de Git:

  1. Añado los archivos al área de staging con git add.
  2. Guardo los cambios con git commit -m "mensaje".
  3. Conecto mi proyecto a GitHub con git remote add origin URL.
  4. Subo los cambios usando git push.

Comandos de Git que he usado:

  • git status: muestra el estado de los archivos.
  • git add: añade archivos al área de preparación (staging).
  • git commit -m "mensaje": guarda los cambios con un mensaje.
  • git push: envía los cambios al repositorio remoto.
  • git clone URL: clona un repositorio remoto.
  • git pull: actualiza mi repositorio con los últimos cambios del remoto.

Comandos de Terminal que he aprendido

Comandos de navegación

  • pwd: muestra el directorio actual.
  • ls: lista los archivos y carpetas del directorio actual.
  • cd [directorio]: cambia al directorio especificado.
    • cd ..: sube un nivel en la jerarquía de carpetas.
    • cd ~: va a mi directorio personal (home).
  • mkdir [directorio]: crea una nueva carpeta.
  • rmdir [directorio]: elimina una carpeta vacía.
  • rm -rf [directorio]: elimina una carpeta y todo su contenido de manera forzada.

Manuales de ayuda

Para conocer más sobre un comando, utilizo:

  • man [comando]: muestra el manual del comando.
  • [comando] --help: muestra una ayuda rápida sobre cómo usarlo.

Comandos de gestión de archivos

  • touch [archivo]: crea un archivo vacío con ese nombre.
  • cp [origen] [destino]: copia el archivo de origen al destino.
  • mv [origen] [destino]: mueve o renombra el archivo.
  • rm [archivo]: elimina un archivo.
