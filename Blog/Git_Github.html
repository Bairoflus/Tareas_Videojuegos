Git es una herramienta que utilizo para controlar versiones de mis proyectos. GitHub me permite compartir esos proyectos y colaborar con otras personas en línea.

Mi flujo de trabajo con Git:

  1. Añado los archivos al área de staging con git add.
  2. Verifico el estado de mis archivos con git status.
  3. Guardo los cambios con git commit -m "mensaje".
  4. Conecto mi proyecto a GitHub con git remote add origin URL.
  5. Subo los cambios usando git push.
  6. Si quiero trabajar en un proyecto existente, lo clono con git clone URL.
  7. Cuando hay cambios nuevos en el repositorio remoto, uso git pull para actualizarlos en mi repositorio local.

Comandos de Git que he usado:

  • git status: muestra el estado de los archivos.
  • git add: añade archivos al área de preparación (staging).
  • git commit -m "mensaje": guarda los cambios con un mensaje.
  • git push: envía los cambios al repositorio remoto.
  • git clone URL: clona un repositorio remoto.
  • git pull: actualiza mi repositorio con los últimos cambios del remoto.
